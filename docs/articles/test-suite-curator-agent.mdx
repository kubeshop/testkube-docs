# Testkube Test Suite Curator Agent

An AI Agent in Testkube can be set up to audit your test suite and identify stale, redundant, or
neglected workflows that need attention — helping you keep your test estate lean and maintainable.

## Requirements

A Test Suite Curator Agent requires:

- **Access to workflow definitions and execution metrics** — provided by the integrated Testkube MCP Server.

This agent uses only the built-in Testkube MCP tools and does not require any external MCP servers.

Optionally, it can also be set up to:
- Run on a schedule via [AI Agent Triggers](/articles/ai-triggers) for periodic test suite health reports
- Focus on specific workflow groups using labels or resource groups

## Create the Test Suite Curator AI Agent

Create an AI Agent as described at [Creating an AI Agent](/articles/defining-ai-agents#creating-a-new-ai-agent), name it
what you want and set the prompt to the following (feel free to adapt it to your needs!):

> You are an AI agent that audits and curates a test suite. Analyze all workflows to find issues
> that affect maintainability, reliability, and efficiency of the test estate.
>
> When asked to audit the test suite:
> 1. Use list_workflows to get all workflows (or a filtered subset by labels/resourceGroup)
> 2. For each workflow, use get_workflow_metrics to check execution frequency, pass rates, and health
> 3. Use list_executions to find workflows with no recent runs (stale workflows)
> 4. Use query_workflows to compare workflow structures and find potential duplicates or overlaps
> 5. Use get_workflow_definition to inspect workflow config for best-practice violations
>
> Identify and report:
> - **Stale workflows**: no executions in the last 30 days (or user-specified threshold)
> - **Permanently failing**: workflows with 0% pass rate over their last N executions — likely abandoned or broken
> - **Duplicate coverage**: workflows testing the same target with similar steps or images
> - **Best-practice violations**: unpinned image tags, missing timeouts, no artifact collection,
>   missing resource requests, no labels
> - **Low-value workflows**: very simple workflows that always pass and may not be testing anything meaningful
>
> For each finding, provide: the workflow name, the issue category, evidence (metrics/config),
> and a recommended action (archive, fix, merge, or keep as-is with explanation).
>
> Present findings grouped by category with a summary count at the top.

Enable the following Testkube MCP tools for this agent:

- `list_workflows` — to discover all workflows
- `get_workflow_metrics` — for execution statistics and health scores
- `get_workflow_definition` — to inspect workflow configuration
- `list_executions` — to check for recent activity
- `query_workflows` — to compare workflow structures and find patterns
- `list_labels` — to understand how workflows are organized

## Using the Test Suite Curator AI Agent

Start a chat with the agent and ask it to audit your test suite:

- *"Audit my test suite and identify workflows that need attention"*
- *"Which workflows haven't run in the last month?"*
- *"Find workflows that are always failing and might be abandoned"*
- *"Check all workflows for best-practice violations"*

### As a Scheduled Trigger

Set up an [AI Agent Trigger](/articles/ai-triggers) to run a periodic audit (e.g. weekly) with a prompt like:

> Audit all workflows for stale tests (no runs in 14 days), permanently failing workflows, and best-practice
> violations. Provide a categorized summary with recommended actions for each finding.
