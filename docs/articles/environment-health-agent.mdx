# Testkube Environment Health Auditor Agent

A periodic health check agent that monitors the overall health of your testing environment — checking
execution error rates, identifying stuck or abnormally long executions, analyzing agent connectivity,
and producing a summary report.

## Requirements

An Environment Health Auditor Agent requires:

- **Access to execution data, workflow metrics, and environment metadata** — provided by the integrated Testkube MCP Server.

This agent uses only the built-in Testkube MCP tools and does not require any external MCP servers.

Optionally, it can also be set up to:
- Run on a schedule via [AI Agent Triggers](/articles/ai-triggers) for daily or weekly health reports
- Abort stuck executions using `abort_workflow_execution` (with approval)

## Create the Environment Health Auditor AI Agent

Create an AI Agent as described at [Creating an AI Agent](/articles/defining-ai-agents#creating-a-new-ai-agent), name it
what you want and set the prompt to the following (feel free to adapt it to your needs!):

> You are a testing environment health monitor. Produce a comprehensive health report for the current
> Testkube environment.
>
> When asked to audit health:
> 1. Use list_agents to check agent connectivity and status
> 2. Use query_executions to find recent executions and calculate error rates
> 3. Use list_executions to identify any currently running or queued executions that may be stuck
> 4. Use get_workflow_metrics across workflows to identify declining health scores
> 5. Use list_workflows to get the total workflow count and check for workflows that have not run recently
>
> Report on the following health dimensions:
>
> **Agent Health**:
> - Number of connected agents and their status
> - Any agents that appear disconnected or unhealthy
>
> **Execution Health** (over the last 24 hours or specified period):
> - Total executions, pass rate, and failure rate
> - Failure rate trend compared to the previous period
> - Any executions that have been running for an unusually long time (potentially stuck)
>
> **Workflow Health**:
> - Workflows with declining health scores
> - Workflows with 0% pass rate (consistently failing)
> - Workflows that have not executed in an unusually long time (potentially abandoned)
>
> **Resource Health**:
> - Workflows showing increasing resource usage trends
> - Any recent OOM kills or resource-related failures
>
> Present the report as a structured health dashboard with:
> - An overall health score (healthy / degraded / critical)
> - A summary of key findings and recommended actions
> - Detailed sections for each health dimension

Enable the following Testkube MCP tools for this agent:

- `list_agents` — to check agent connectivity and status
- `query_executions` — to analyze recent execution patterns
- `list_executions` — to find running/queued/stuck executions
- `get_workflow_metrics` — for workflow health scores and statistics
- `list_workflows` — to discover all workflows and check activity
- `query_workflows` — to filter and search workflows
- `get_workflow_resource_history` — to check for resource usage trends
- `list_labels` — to understand environment organization
- `abort_workflow_execution` — to abort stuck executions if needed (requires approval)

## Using the Environment Health Auditor AI Agent

### As a Scheduled Report

Set up an [AI Agent Trigger](/articles/ai-triggers) to generate a daily health report:

- **Trigger Events**: Test Workflow Success (use any frequently running workflow as the "clock")
- **Trigger Mode**: On state change
- **Prompt Template**:
  ```
  Produce a daily health report for this environment.
  Cover agent health, execution error rates for the last 24 hours,
  stuck executions, and workflows with declining health scores.
  ```

:::tip
For a true scheduled health check, use the [Scheduled AI Agent Trigger](/articles/ai-triggers#scheduled-ai-agent-triggers)
pattern — create a lightweight scheduler workflow on a cron schedule and attach this trigger to it.
The agent's analysis covers the entire environment, not just the triggering workflow.
:::

### Interactive Analysis

Start a chat with the agent for on-demand health checks:

- *"Give me a health report for this environment"*
- *"Are there any stuck executions right now?"*
- *"Which workflows have the worst pass rates this week?"*
- *"Is my environment healthy enough for a release?"*
