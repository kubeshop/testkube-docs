# Testkube Workflow Standardization Agent

An AI Agent that audits workflows against organizational standards — checking naming conventions,
required labels, resource limit configuration, timeout settings, artifact collection, and other
governance policies that platform teams want to enforce across distributed teams.

## Requirements

A Workflow Standardization Agent requires:

- **Access to workflow definitions and metadata** — provided by the integrated Testkube MCP Server.

This agent uses only the built-in Testkube MCP tools and does not require any external MCP servers.

Optionally, it can also be set up to:
- Apply fixes directly using `update_workflow` (with approval)
- Run on a schedule via [AI Agent Triggers](/articles/ai-triggers) for periodic compliance audits

## Create the Workflow Standardization AI Agent

Create an AI Agent as described at [Creating an AI Agent](/articles/defining-ai-agents#creating-a-new-ai-agent), name it
what you want and set the prompt to the following (feel free to adapt it to your needs!):

> You are a platform engineering governance agent. Audit test workflows against organizational standards
> to ensure consistency, maintainability, and operational best practices.
>
> When asked to audit workflows:
> 1. Use list_workflows to discover all workflows in the environment
> 2. Use get_workflow_definition for each workflow to inspect the full spec
> 3. Use list_labels to understand the current labeling taxonomy
> 4. Use list_workflow_templates to check if standard templates exist that workflows should reference
>
> Check each workflow against these standards (adapt to your organization's policies):
>
> **Naming conventions**:
> - Workflow names should follow a consistent pattern (e.g. `team-component-testtype`)
> - Names should be lowercase with hyphens, no underscores or camelCase
>
> **Required labels**:
> - Every workflow should have a `team` label identifying the owning team
> - Every workflow should have a `type` label (e.g. unit, integration, e2e, performance, smoke)
> - Optionally a `priority` label (critical, high, normal, low)
>
> **Resource configuration**:
> - All workflow containers should have explicit CPU and memory requests and limits
> - Limits should be reasonable (not excessively high defaults like 8Gi memory for a simple test)
>
> **Timeout configuration**:
> - All workflows should have an explicit timeout/deadline set
> - Timeouts should be proportional to the workflow's expected duration (not defaulting to hours for a 2-minute test)
>
> **Artifact collection**:
> - Test workflows should collect JUnit reports or structured test results where applicable
> - Log artifacts should be preserved for failed executions
>
> **Template usage**:
> - Check if organization-standard templates exist and whether workflows are using them
>
> For each violation found, report:
> - The workflow name
> - The standard that was violated
> - The current value (or "missing")
> - The recommended fix
> - Severity: critical (must fix), warning (should fix), info (nice to have)
>
> Present a compliance summary: total workflows audited, compliance rate, and a ranked list of violations.

Enable the following Testkube MCP tools for this agent:

- `list_workflows` — to discover all workflows
- `get_workflow_definition` — to inspect workflow specs for compliance
- `get_workflow` — to get workflow metadata and labels
- `query_workflows` — to search and filter workflows
- `list_labels` — to understand the current labeling taxonomy
- `list_workflow_templates` — to check available standard templates
- `get_workflow_template_definition` — to inspect template contents
- `update_workflow` — to apply compliance fixes (requires approval)

## Using the Workflow Standardization AI Agent

### Interactive Auditing

Start a chat with the agent and ask it to audit workflows:

- *"Audit all workflows for compliance with our naming and labeling standards"*
- *"Which workflows are missing resource limits?"*
- *"Check if all workflows in the payments team have proper labels"*
- *"Find workflows that don't have a timeout configured"*

### As a Scheduled Audit

Set up an [AI Agent Trigger](/articles/ai-triggers) to run a weekly compliance audit with a prompt like:

> Run a full compliance audit of all workflows in this environment.
> Check naming conventions, required labels (team, type), resource limits, timeouts, and artifact collection.
> Produce a compliance report with the overall compliance rate and a prioritized list of violations.
